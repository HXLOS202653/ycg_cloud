# 种子数据说明文档

## 概述

种子数据脚本用于初始化YCG云盘系统的基础数据，包括管理员用户、系统配置、默认团队和文件夹结构等。

## 文件说明

### 核心迁移文件

- `20250112120000_create_seed_data.up.sql` - 种子数据创建脚本
- `20250112120000_create_seed_data.down.sql` - 种子数据回滚脚本

### 辅助管理脚本

- `../scripts/seed_data_manager.sql` - 种子数据管理工具（存储过程）

- **系统基础配置**：应用名称、版本、维护模式等

- **文件管理配置**：文件大小限制、允许扩展名、分片大小等
- **分享配置**：默认过期时间、密码策略等
- **通知配置**：邮件、浏览器通知开关等
- **OSS配置**：对象存储服务配置（需要修改为实际值）

- 系统管理员：根目录、系统文档、备份文件

## 使用方法

### 开发环境

```bash
# 应用种子数据
go run cmd/migration/main.go up

# 或者直接执行SQL
mysql -u root -p ycg_cloud < migrations/mysql/20250112120000_create_seed_data.up.sql
```

### 生产环境

1. **修改密码**：在执行前修改脚本中的所有密码哈希值
2. **修改OSS配置**：更新为实际的对象存储配置
3. **修改邮箱**：更新为实际的管理员邮箱
4. **执行迁移**：通过迁移工具执行

```bash
# 生产环境谨慎执行
go run cmd/migration/main.go up
```

### 回滚种子数据

⚠️ **危险操作**：此操作会删除所有种子数据

```bash
# 回滚种子数据
go run cmd/migration/main.go down

# 或者直接执行SQL
mysql -u root -p ycg_cloud < migrations/mysql/20250112120000_create_seed_data.down.sql
```

## 安全建议

### 密码安全

1. **立即修改默认密码**：所有默认用户的密码都必须修改
2. **使用强密码**：至少8位，包含大小写字母、数字和特殊字符
3. **启用双因子认证**：为管理员账户启用2FA
4. **定期更换密码**：建议每3-6个月更换一次

### 配置安全

1. **修改OSS密钥**：使用真实的OSS访问密钥
2. **检查敏感配置**：确保敏感配置的访问权限正确
3. **定期审计**：定期检查系统配置的变更

### 数据安全

1. **备份种子数据**：在修改前备份原始种子数据
2. **测试环境验证**：先在测试环境验证种子数据
3. **监控日志**：监控种子数据相关的操作日志

## 故障排除

### 常见问题

1. **外键约束错误**：确保按正确顺序执行迁移
2. **重复数据错误**：检查是否已存在相同的用户名或邮箱
3. **权限错误**：确保数据库用户有足够的权限

### 解决方案

1. **重置自增ID**：如果需要重新开始，可以重置表的自增ID
2. **强制清理**：使用`CleanTestData`存储过程清理测试数据
3. **手动修复**：必要时手动调整数据关系

## 维护建议

1. **定期检查**：定期运行`CheckSeedDataStatus`检查数据状态
2. **清理测试数据**：定期清理测试环境的临时数据
3. **更新配置**：根据业务需求及时更新系统配置
4. **监控使用**：监控管理员账户的使用情况
