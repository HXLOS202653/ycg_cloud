# 网络云盘系统 - 命名规范

## 📋 概述

统一的命名规范是保证代码质量和团队协作效率的重要基础。本文档基于网络云盘系统的技术栈（React 18 + TypeScript + Go + Gin + GORM + MySQL + MongoDB + Redis + MySQL全文索引）制定了前端、后端、数据库、API、文档等各个层面的命名标准，确保整个项目的命名风格一致、可读性强、易于维护。

### 技术栈对应的命名约定
- **前端**: React 18 + TypeScript + Ant Design + Redux Toolkit
- **后端**: Go 1.24.6 + Gin 1.10.0 + GORM 1.30.1 + WebSocket
- **数据库**: MySQL 8.0 + MongoDB 6.0 + Redis 7.0+ + MySQL全文索引
- **消息队列**: Redis Pub/Sub + Stream 7.0+
- **部署**: Docker + Kubernetes + Nginx
- **监控**: Prometheus + Grafana + ELK Stack


**命名一致性检查清单**：
```sql
-- ✅ 正确的跨数据库字段命名
-- MySQL表
CREATE TABLE files (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,  -- 主键统一命名为id
    user_id BIGINT UNSIGNED NOT NULL,              -- 外键统一为{表名}_id
    file_name VARCHAR(255) NOT NULL                -- 字段名使用snake_case
);

-- MongoDB集合
db.file_analysis.insertOne({
    file_id: NumberLong(123),                      -- 关联字段：{表名}_id
    user_id: NumberLong(456),                      -- 外键字段：{表名}_id
    analysis_type: "content"                       -- MongoDB字段使用snake_case
});

-- ❌ 错误示例
-- 字段名不一致
files.id ↔ file_analysis.fileId               # MongoDB应使用file_id
users.id ↔ user_preferences.userId            # MongoDB应使用user_id

-- 类型不匹配
files.id (BIGINT) ↔ file_analysis.file_id (String)    # 类型不匹配
```

### 5. 数据库索引命名统一规范

```sql
-- ✅ MySQL索引命名（已有规范）
CREATE INDEX idx_user_email ON users(email);
CREATE INDEX idx_file_user_parent ON files(user_id, parent_id);
CREATE UNIQUE INDEX uk_session_token ON user_sessions(session_token);

-- ✅ MongoDB索引命名（新增统一规范）
db.file_analysis.createIndex(
    { "file_id": 1 }, 
    { name: "idx_file_id" }                        -- 统一使用idx_前缀
);

db.file_analysis.createIndex(
    { "file_id": 1, "analysis_type": 1 }, 
    { 
        unique: true, 
        name: "idx_file_id_analysis_type_unique"    -- 复合索引：字段名_unique
    }
);

db.user_preferences.createIndex(
    { "user_id": 1 }, 
    { 
        unique: true, 
        name: "idx_user_id_unique"                  -- 唯一索引：字段名_unique
    }
);

-- ❌ 错误示例 - MongoDB索引命名不规范
db.file_analysis.createIndex({ "file_id": 1 });    # 缺少name参数
db.file_analysis.createIndex(                      # 索引名不符合规范
    { "file_id": 1 },
```

## 📋 总结和最佳实践

### 命名原则优先级
1. **一致性** > **简洁性** > **描述性**
2. **可读性** > **简短性**
3. **标准化** > **个人喜好**
4. **国际化** > **本地化**

### 不同语言命名风格对照表

| 类型 | JavaScript/TypeScript | Go | SQL | JSON API |
|------|----------------------|----|----- |----------|
| 变量 | camelCase | camelCase | snake_case | snake_case |
| 函数 | camelCase | PascalCase(公开)<br>camelCase(私有) | snake_case | snake_case |
| 常量 | SCREAMING_SNAKE_CASE | PascalCase | SCREAMING_SNAKE_CASE | snake_case |
| 类/结构体 | PascalCase | PascalCase | snake_case | snake_case |
| 包/模块 | kebab-case | lowercase | snake_case | kebab-case |
| 文件 | kebab-case | snake_case | snake_case | kebab-case |
