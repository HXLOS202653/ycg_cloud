# 网络云盘系统 - 完整数据库设计文档

## 📋 设计概述

### 数据库架构
本系统采用**多数据库架构**，根据不同数据特性选择最适合的存储方案：

- **MySQL 8.0.31**: 存储结构化数据（用户、文件、权限、团队等）
- **MongoDB 6.0**: 存储半结构化数据（聊天消息、AI分析结果、日志等）
- **Redis 7.0+**: 缓存热点数据、消息队列（会话、上传进度、实时状态等）
- **MySQL 8.0+ 全文索引**: 全文搜索引擎（文件内容索引、搜索日志等）

### 设计原则
- **数据完整性**: 严格的外键约束和数据验证
- **性能优化**: 合理的索引设计和分区策略
- **扩展性**: 预留扩展字段，支持业务增长
- **安全性**: 敏感数据加密，权限严格控制
- **一致性**: 事务保证数据一致性
- **命名规范**: 严格遵循项目命名规范

### 技术栈版本
- **MySQL**: 8.0.31 (InnoDB引擎，支持全文索引)
- **MongoDB**: 6.0 LTS (WiredTiger存储引擎，长期支持版本)
- **Redis**: 7.0+ (支持Stream、Pub/Sub、模块和集群)
- **MySQL全文索引**: 8.0+ (支持中文ngram解析器)

**回滚操作流程**：
```go
// 1. 回滚单个事务
func RollbackTransaction(transactionID string) error {
    // 查询事务的所有操作记录
    operations, err := mysql.GetOperationsByTxID(transactionID)
    if err != nil {
        return err
    }
    
    // 按时间倒序回滚操作
    sort.Slice(operations, func(i, j int) bool {
        return operations[i].CreatedAt.After(operations[j].CreatedAt)
    })
    
    for _, op := range operations {
        switch op.Operation {
        case "INSERT":
            // 删除插入的记录
            err = deleteRecord(op.TableName, op.RecordID)
        case "UPDATE":
            // 恢复到修改前的数据
            err = restoreRecord(op.TableName, op.RecordID, op.BeforeData)
        case "DELETE":
            // 恢复被删除的记录
            err = insertRecord(op.TableName, op.BeforeData)
        }
        
        if err != nil {
            log.Errorf("回滚操作失败: %v", err)
            continue
        }
        
        // 标记操作已回滚
        mysql.UpdateOperationStatus(op.ID, "ROLLEDBACK")
    }
    
    return nil
}

// 2. 批量回滚机制
func BatchRollback(criteria RollbackCriteria) error {
    // 根据条件查找需要回滚的事务
    transactions, err := mysql.GetTransactionsForRollback(criteria)
    if err != nil {
        return err
    }
    
    // 并发回滚多个事务
    semaphore := make(chan struct{}, 5) // 限制并发度
    for _, txID := range transactions {
        go func(id string) {
            semaphore <- struct{}{}
            defer func() { <-semaphore }()
            
            if err := RollbackTransaction(id); err != nil {
                log.Errorf("事务回滚失败 %s: %v", id, err)
            }
        }(txID)
    }
    
    return nil
}
```

func ConsistencyCheck() error {
    // 1. 检查MySQL与MongoDB的关联数据
    inconsistentFiles, err := checkFileConsistency()
    if err != nil {
        return err
    }
    
    if len(inconsistentFiles) > 0 {
        log.Warnf("发现数据不一致文件: %v", inconsistentFiles)
        
        // 计算不一致比例
        totalFiles, _ := mysql.CountTotalFiles()
        inconsistencyRate := float64(len(inconsistentFiles)) / float64(totalFiles)
        
        // 超过容忍度触发修复
        if inconsistencyRate > config.ConsistencyTolerance {
            return triggerDataRepair(inconsistentFiles)
        }
    }
    
    return nil
}

func triggerDataRepair(inconsistentFiles []FileInconsistency) error {

### 4. 缓存策略

#### 4.1 缓存更新策略详细设计

**Cache-Aside模式实现**：
```go
// 读取数据时的缓存逻辑
func GetFileInfo(fileID int64) (*FileInfo, error) {
    // 1. 先查缓存
    cacheKey := fmt.Sprintf("file:info:%d", fileID)
    cached, err := redis.Get(cacheKey)
    if err == nil {
        return parseFileInfo(cached), nil
    }
    
    // 2. 缓存未命中，查数据库
    fileInfo, err := mysql.GetFileInfo(fileID)
    if err != nil {
        return nil, err
    }
    
    // 3. 数据写入缓存
    redis.SetEX(cacheKey, serialize(fileInfo), 30*time.Minute)
    return fileInfo, nil
}

// 更新数据时的缓存逻辑
func UpdateFileInfo(fileID int64, info *FileInfo) error {
    // 1. 先更新数据库
    err := mysql.UpdateFileInfo(fileID, info)
    if err != nil {
        return err
    }
    
    // 2. 删除缓存（避免缓存不一致）
    cacheKey := fmt.Sprintf("file:info:%d", fileID)
    redis.Del(cacheKey)
    
    // 3. 可选：立即回写缓存（对于热点数据）
    redis.SetEX(cacheKey, serialize(info), 30*time.Minute)
    return nil
}
```

func GetFileInfoWithLock(fileID int64) (*FileInfo, error) {
    cacheKey := fmt.Sprintf("file:info:%d", fileID)
    lockKey := fmt.Sprintf("lock:file:info:%d", fileID)
    
    // 1. 查询缓存
    cached, err := redis.Get(cacheKey)
    if err == nil {
        return parseFileInfo(cached), nil
    }
    
    // 2. 获取分布式锁
    lock, err := redis.SetNX(lockKey, "1", 10*time.Second)
    if err != nil || !lock {
        // 获取锁失败，等待并重试
        time.Sleep(100 * time.Millisecond)
        return GetFileInfoWithLock(fileID) // 递归重试
    }
    
    defer redis.Del(lockKey) // 释放锁
    
    // 3. 双重检查缓存
    cached, err = redis.Get(cacheKey)
    if err == nil {
        return parseFileInfo(cached), nil
    }
    
    // 4. 查询数据库并写入缓存
    fileInfo, err := mysql.GetFileInfo(fileID)
    if err == nil {
        redis.SetEX(cacheKey, serialize(fileInfo), 30*time.Minute)
    }
    
    return fileInfo, err
}
